GATE 2.1 — APPROVAL-REVIEW CORRECTIONS APPLIED
1 · Today hierarchy: identity → food status → next event → compact garden (never leading); no default «ثبت غذای امروز»; empty day is read-only (Flow B).
2 · Commerce timing: a paid order is «در راه» — planned assignment only; the estimate starts at confirmed opening, never at delivery (A5–A7, C, S).
3 · Approved content only: unapproved journeys/guides are hidden or disabled — «در انتظار تأیید» is internal-only (E2b, C4).
4 · Estimation honesty: owner-stated shares & mix, «نمی‌دانم» → household fallback + wider bands; reorders are never consumption evidence (C5–C8).
5 · Garden: eligibility-based milestone catalogue, illustrative (not scheduled) 90-day example, recap → memory card only, explicit quadrant rule (§5).
6 · Reorder explanation: customer outcome first; arithmetic behind optional «چطور محاسبه شد؟» (D2, D2b).
7 · سفر سفارش: plain status flow (Flow S) replaces the deprecated Journey Caravan artifact.
