5 · Finalized components, responsive behavior & engineering annotations

PetSwitcher — pets[], activeId, onSwitch · avatar disc + name chip, 44px; overflow ≥ 4 pets scrolls horizontally, active chip pinned.
FoodModule — state: incoming | active | unavailable | error | setup · owns the second slot of Today; never renders days-remaining in incoming/setup.
MeterBand — range, marker, confidence: high|mid|unknown · striped fill iff household-level; requires provenance rows.
StatusChip — glyph + word + optional date; tones pos/info/warn/mute; color never sole channel.
EventCard — source: owner | journey | system · journey cards show «روز X از Y»; max one actionable per day.
GardenStrip — height locked 76px; preview-only; tap → GardenStage.
GardenStage — quadrant plan, slots[], objects[] (each linked memoryId), presence loop; placement via drag or tap.

QuarterScale — پر / بیش از نصف / کمتر از نصف / ته کیسه — setup and correction share this one control.
RESPONSIVE BEHAVIOR
320–359px — screen padding 16; meter ring 56px; pet chips text-only after 3 pets; type scale unchanged (floor 13.5px body).
360–430px (reference) — layouts as drawn; screen padding 20.
431–599px — content max-width 430 centered; Garden stage grows to fill.
≥ 600px (tablet) — Today becomes two columns: identity+food+event right, Garden preview + diary teasers left (RTL); sheets become centered dialogs (max 480px); Garden stage caps at 560px with the object list beside it.
Landscape / keyboard — sheets never exceed 70% height; content scrolls inside, actions pinned.
RTL: logical properties only; Persian digits everywhere customer-facing; Latin codes isolated dir="ltr"; ZWNJ compounds required in all copy.
ENGINEERING ANNOTATIONS
Estimate lifecycle — estimate object created ONLY by a confirmed-opening event; delivery events mutate inventory, never estimates. Reorders are never consumption evidence.
Confidence — derived from input provenance (recorded ● > owner-stated > unknown) + staleness; maps to band width + word + texture. Never expose a raw score.
Content gating — care-journey plans, guide values and equivalence claims carry an approval flag; unapproved → component renders nothing (no pending state exists client-side).
Today composition — modules load independently; identity from cache first; per-module error boundaries (B4); skeletons at final geometry.
Garden economy — unlocks are server-side eligibility events referencing the milestone catalogue; each object stores memoryId; no client-side counters, no XP field anywhere in the schema.
Notifications — reorder card mirrors to at most one push; snooze suppresses both for 72h unless the pessimistic bound worsens.
Policy placeholders — delay compensation, cancellation-after-sourcing, refunds, replacement: components exist behind flags, ship disabled until Product/Ops constants land.
